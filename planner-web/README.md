# React + TypeScript + Vite + shadcn/ui

This is a template for a new Vite project with React, TypeScript, and shadcn/ui.

## TanStack Router Nested File Routes

This project uses TanStack Router with file-based routing. When a route has children, the parent route file must render an `<Outlet />` component for child routes to appear.

### Key Conventions

- **Parent layout routes**: When a route directory contains child routes (e.g., `$itemId.tsx` with `complete.tsx` inside `$itemId/`), the parent file (`$itemId.tsx`) must be a layout route that imports `Outlet` from `@tanstack/react-router` and returns `<Outlet />` in its component.

- **Index routes**: Concrete page content at the same URL level as a parent with children belongs in an `index.tsx` file. For example, the item detail page at `/plans/$planId/payment-periods/$periodId/items/$itemId` lives in `index.tsx`, while `$itemId.tsx` is the layout route.

- **Route path for index files**: Index route files use a trailing slash in the path:
  ```tsx
  export const Route = createFileRoute("/plans/$planId/payment-periods/$periodId/items/$itemId/")({
    component: MyPage,
  })
  ```

- **Generated route tree**: `src/routeTree.gen.ts` is automatically generated by `@tanstack/router-plugin/vite` when running `pnpm dev` or `pnpm build`. Never edit this file by hand—it will be overwritten.

### Example Structure

```
routes/plans/$planId/payment-periods/$periodId/items/$itemId/
├── index.tsx          # Item detail page (matches /plans/$planId/payment-periods/$periodId/items/$itemId/)
├── complete.tsx       # Complete item page (matches /plans/$planId/payment-periods/$periodId/items/$itemId/complete)
└── $itemId.tsx        # Layout route (renders <Outlet />)
```

In this structure, `$itemId.tsx` is the parent layout that renders `<Outlet />`, `index.tsx` renders the item detail page when visiting `/plans/$planId/payment-periods/$periodId/items/$itemId/`, and `complete.tsx` renders when visiting `/plans/$planId/payment-periods/$periodId/items/$itemId/complete`.

## Adding components

To add components to your app, run the following command:

```bash
npx shadcn@latest add button
```

This will place the ui components in the `src/components` directory.

## Using components

To use the components in your app, import them as follows:

```tsx
import { Button } from "@/components/ui/button"
```
